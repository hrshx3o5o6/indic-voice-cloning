## IndicF5 Model API (ai4bharat/IndicF5)

Based on HuggingFace model card for ai4bharat/IndicF5:
- Model type: AutoModelForCausalLM (pretrained transformers model)
- Tokenizer: AutoTokenizer from same repo
- Input: text string (Indic language), reference audio path, reference transcript
- Output: Generated audio waveform (numpy array, typically int16 or float32)
- Inference: Model has a generate() or inference() method
- Device: Supports cuda, cpu; MPS explicitly broken (vocos ISTFT crash on Apple M4)
- Checkpoint size: 1.4 GB (downloaded on first run to ~/.cache/huggingface)

Key imports needed:
```python
from transformers import AutoModelForCausalLM, AutoTokenizer
import torch
import torchaudio
import numpy as np
import soundfile as sf
```

## Audio I/O Patterns

From existing codebase (tts_sarvam.py, test_tts_sarvam.py):
- Output format: WAV file at output_path
- Use soundfile library for writing: `soundfile.write(output_path, audio_data, sample_rate)`
- Audio normalization: If int16 (range -32768 to 32767), convert to float32 (range -1.0 to 1.0) by dividing by 32768.0

From test fixtures (conftest.py):
- WAV generation: Use wave module to create test WAV files
- Sample rate: 22050 Hz (Sarvam default); IndicF5 should use its model's native sample rate
- Channels: 1 (mono)

From INDICF5-01 signature: ref_audio_path is a file path
- Load audio from disk using torchaudio.load()
- Audio should be resampled to match model's expected sample rate (check IndicF5 docs)
- Pass to model as context for voice cloning

After model and tokenizer are loaded, add inference:

```python
    # Tokenize input text
    input_ids = tokenizer(text, return_tensors="pt").input_ids.to(device)
    
    # Generate speech conditioned on reference audio and transcript
    # Note: Exact inference signature may differ; check IndicF5 docs for exact method
    try:
        with torch.no_grad():
            output = model.generate(
                input_ids,
                max_length=1024,  # Adjust based on expected output length
                do_sample=True,
                top_p=0.95,
                temperature=0.7,
                # Reference audio context: may need special handling per model docs
            )
        # Output is typically a waveform tensor; convert to numpy
        if isinstance(output, torch.Tensor):
            audio_data = output.cpu().numpy()
        else:
            audio_data = output
    except Exception as e:
        raise RuntimeError(f"IndicF5 inference failed: {e}")
    
    # Normalize audio: int16 → float32 (per INDICF5-07)
    if audio_data.dtype == np.int16:
        audio_data = audio_data.astype(np.float32) / 32768.0
    elif audio_data.dtype != np.float32:
        audio_data = audio_data.astype(np.float32)
    
    # Write output WAV file
    try:
        # Determine sample rate (use model's native rate or ref_audio rate)
        output_sample_rate = 16000  # Default; check IndicF5 docs for actual rate
        sf.write(output_path, audio_data, output_sample_rate)
    except Exception as e:
        raise RuntimeError(f"Failed to write output audio: {e}")
    
    return output_path
```

**Key points:**
- Tokenize input text matching the language
- Inference with torch.no_grad() to avoid gradients (inference only)
- Handle both tensor and numpy output from model
- **CRITICAL**: Int16 → float32 normalization (per INDICF5-07): divide by 32768.0 to map [-32768, 32767] → [-1.0, 1.0]
- Use soundfile.write() to output WAV (matches existing pattern from tts_sarvam.py)
- Sample rate should match model's output or reference audio rate (verify in IndicF5 docs)
- Return output_path for easy chaining (per INDICF5-01)
