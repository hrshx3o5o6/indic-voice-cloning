## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| File system → output_path | External path provided by user; must be validated as writable before inference |
| Network → HuggingFace checkpoint download | Checkpoint download happens via huggingface_hub; MitM attacks possible but standard trust model for open-source models |
| ref_audio_path file → IndicF5 model input | Audio file content untrusted; malformed WAV should raise clear error, not crash |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | ref_audio file | mitigate | Validate file exists and is readable before passing to model loader; catch IOError and re-raise as FileNotFoundError with clear message |
| T-02-02 | Information Disclosure | HF checkpoint download | accept | Standard open-source risk; model weights are public; no PII in checkpoint path |
| T-02-03 | Denial of Service | output_path creation | accept | User supplies path; if invalid, soundfile will raise IOError (caught by test in 02-03) |
| T-02-04 | Elevation of Privilege | Model inference on GPU | accept | Model runs in user's Python process with user's permissions; no privilege escalation possible |
</threat_model>
